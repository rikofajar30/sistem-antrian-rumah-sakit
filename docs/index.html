Sistem Antrian LAN Terdistribusi RS Waluyo (Anti-Blackout Version)

1. Gambaran Umum Sistem

Sistem Antrian LAN Terdistribusi RS Waluyo dirancang khusus untuk mengotomatisasi seluruh proses administrasi penomoran antrian pasien di poliklinik secara real-time tanpa menggunakan koneksi internet luar (murni menggunakan jaringan intranet / LAN).

Tujuan dan Manfaat

  • Tujuan: Mempercepat alur pelayanan loket rumah sakit dan mengintegrasikan antrian lintas poliklinik secara terpusat.
  • Manfaat: Menghilangkan penumpukan antrean fisik, meningkatkan transparansi pameran nomor lewat TV monitor ruang tunggu, serta mempermudah dokter memanggil pasien jarak jauh via HP pribadi.

Target Pengguna

  1. Pasien: Pengambil nomor tiket mandiri pada unit kios di lobi utama.
  2. Dokter / Perawat: Pemegang kendali panggilan antrian poliklinik melalui dashboard HP medis.
  3. Pengunjung Umum: Pemantau status antrian pada layar TV utama di ruang tunggu.

Fitur Unggulan

  • Kios Tiket Multi-Poli: Penomoran otomatis berbasis prefiks poli (Umum: A, Gigi: B, Anak: C).
  • Real-Time WebSocket Sync: Transmisi data instan dua arah antar perangkat memanfaatkan library Socket.io melalui integrasi event data terpusat.
  • Voice Announcement System: Suara bel otomatis (suaraBel) yang ditiupkan langsung dari browser TV ruang tunggu setelah interaksi klik pertama pada layar.
  • Anti-Blackout Storage: Keamanan data tingkat tinggi menggunakan modul fs (File System) Node.js yang menuliskan objek data ke harddisk fisik server (antrian.json) setiap kali ada perubahan pendaftaran maupun panggilan, melindunginya dari risiko mati lampu.
  • Auto-Reset System: Reset berkala data antrian menjadi 000 secara otomatis setiap kali memuat ulang server pada deteksi tanggal hari berikutnya (new Date().toDateString()).

2. Panduan Penggunaan Fitur Secara Terurut

Sistem ini beroperasi mengikuti urutan sekuensial yang saling terintegrasi antar perangkat sebagai berikut:

Langkah 1: Pengambilan Tiket (Kios Pasien)

Pasien menggunakan HP/Tablet di lobi utama yang mengarah ke alamat URL http://IP_SERVER:5500/public/kios.html. Pasien memilih salah satu tombol poliklinik (UMUM, GIGI, atau ANAK). Sistem memicu event ambilTiket, menaikkan counter pendaftar, membuat nomor antrian terformat, memasukkan riwayat ke array history dengan status menunggu, dan mengamankannya ke harddisk.

Langkah 2: Pemantauan Layar Tunggu (TV Monitor)

Petugas menyalakan browser Smart TV di ruang tunggu dan membuka alamat URL http://IP_SERVER:5500/public/monitor.html. Petugas wajib melakukan klik satu kali pada area layar mana saja untuk memicu aktivasi sistem audio bel browser agar dapat mendengarkan siaran suara panggil otomatis.

Langkah 3: Pemanggilan Pasien (HP Dokter)

Dokter membuka HP pribadinya, mengakses portal utama http://IP_SERVER:5500, dan masuk ke Panel Medis dengan data yang tersimpan di localStorage (petugas_user dan petugas_poli). Dokter dapat melacak total antrian di kios lewat box status Total Antrian Diambil Kios. Ketika menekan tombol "Panggil Selanjutnya", server memperbarui status array history menjadi dipanggil, mengubah penunjuk nomor aktif loket, dan memancarkan sinyal pemicu bel suara ke TV monitor ruang tunggu.

Langkah 4: Penyelesaian Sesi

Begitu penanganan medis selesai, dokter menekan tombol "Selesai Penanganan" pada HP miliknya. Sistem akan memicu pembaruan data backend, merubah status tiket menjadi selesai, serta memperbarui nilai counter aktif monitor TV menjadi teks SELESAI sehingga loket siap memproses antrian berikutnya.

3. Spesifikasi Arsitektur Penyimpanan Data Persistent Storage

Data dikunci dalam bentuk file teks terstruktur terpusat di direktori root server dengan nama file antrian.json. Struktur objek disusun sebagai berikut:

{
  "tanggal": "Tue Jun 16 2026",
  "history": [
    {
      "id": 1718548200000,
      "poli": "UMUM",
      "nomor": "A-001",
      "status": "selesai",
      "oleh": "DR. BUDI"
    }
  ],
  "counters": {
    "UMUM": { "current": 1, "active": "SELESAI", "prefix": "A", "petugas": "DR. BUDI" },
    "GIGI": { "current": 0, "active": "000", "prefix": "B", "petugas": "-" },
    "ANAK": { "current": 0, "active": "000", "prefix": "C", "petugas": "-" }
  }
}
