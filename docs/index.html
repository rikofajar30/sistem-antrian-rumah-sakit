Sistem Antrian LAN Terdistribusi RS Waluyo (Anti-Blackout Version)

1. Gambaran Umum Sistem

Sistem Antrian LAN Terdistribusi RS Waluyo dirancang khusus untuk mengotomatisasi seluruh proses administrasi penomoran antrian pasien di poliklinik secara real-time tanpa menggunakan koneksi internet luar (murni menggunakan jaringan intranet / LAN).

Tujuan dan Manfaat

  • Tujuan: Mempercepat alur pelayanan loket rumah sakit dan mengintegrasikan antrian lintas poliklinik secara terpusat.
  • Manfaat: Menghilangkan penumpukan antrean fisik, meningkatkan transparansi pameran nomor lewat TV monitor ruang tunggu, serta mempermudah dokter memanggil pasien jarak jauh via HP pribadi.

Target Pengguna

  1. Pasien: Pengambil nomor tiket mandiri pada unit kios di lobi utama.
  2. Dokter / Perawat: Pemegang kendali panggilan antrian poliklinik melalui dashboard HP medis.
  3. Pengunjung Umum: Pemantau status antrian pada layar TV utama di ruang tunggu.

Fitur Unggulan

  • Kios Tiket Multi-Poli: Penomoran otomatis berbasis prefiks poli (Umum: A, Gigi: B, Anak: C).
  • Real-Time WebSocket Sync: Transmisi data instan dua arah antar perangkat memanfaatkan library Socket.io melalui integrasi event data terpusat (updateData).
  • Voice Announcement System: Suara bel otomatis (suaraBel) yang ditiupkan langsung dari browser TV ruang tunggu setelah interaksi klik pertama pada layar.
  • Anti-Blackout Storage: Keamanan data tingkat tinggi menggunakan modul fs (File System) Node.js yang menuliskan objek data ke harddisk fisik server (antrian.json) setiap kali ada perubahan pendaftaran maupun panggilan, melindunginya dari risiko mati lampu.
  • Auto-Reset System: Reset berkala data antrian menjadi 000 secara otomatis setiap kali memuat ulang server pada deteksi tanggal hari berikutnya (new Date().toDateString()).

2. Panduan Penggunaan Fitur Secara Terurut

Sistem ini beroperasi mengikuti urutan sekuensial yang saling terintegrasi antar perangkat sebagai berikut:

Langkah 1: Pengambilan Tiket (Kios Pasien)

Pasien menggunakan HP/Tablet di lobi utama yang mengarah ke alamat URL http://IP_SERVER:5500/public/kios.html. Pasien memilih salah satu tombol poliklinik (UMUM, GIGI, atau ANAK). Sistem memicu event ambilTiket, menaikkan counter pendaftar (.current), membuat nomor antrian terformat, memasukkan riwayat ke array history dengan status menunggu, dan mengamankannya ke harddisk.

Langkah 2: Pemantauan Layar Tunggu (TV Monitor)

Petugas menyalakan browser Smart TV di ruang tunggu dan membuka alamat URL http://IP_SERVER:5500/public/monitor.html. Petugas wajib melakukan klik satu kali pada area layar mana saja untuk memicu aktivasi sistem audio bel browser agar dapat mendengarkan siaran suara panggil otomatis.

Langkah 3: Pemanggilan Pasien (HP Dokter)

Dokter membuka HP pribadinya, mengakses portal utama http://IP_SERVER:5500, dan masuk ke Panel Medis sesuai poliklinik tugasnya. Sesi login tersimpan otomatis di browser (localStorage.getItem('petugas_poli')). Dokter dapat melacak total antrian di kios lewat box status Total Antrian Diambil Kios. Ketika menekan tombol "Panggil Selanjutnya", server memperbarui status array history menjadi dipanggil, mengubah penunjuk nomor aktif loket, dan memancarkan sinyal pemicu bel suara ke TV monitor ruang tunggu.

Langkah 4: Penyelesaian Sesi

Begitu penanganan medis selesai, dokter menekan tombol "Selesai Penanganan" pada HP miliknya. Sistem akan memicu pembaruan data backend, merubah status tiket menjadi selesai, serta memperbarui nilai counter aktif monitor TV menjadi teks SELESAI sehingga loket siap memproses antrian berikutnya.

3. Panduan Pemasangan dan Konfigurasi Jaringan (Deployment Guide)

Sistem ini menggunakan arsitektur Pusat-Cabang (Server-Client) berbasis jaringan lokal. Node.js hanya perlu diinstal di Satu Laptop (yang bertindak sebagai Server), sedangkan perangkat lain bertindak sebagai pengguna.

Bagian A: Persiapan pada Perangkat Laptop Server (Pusat Data)

  1. Instalasi Runtime Environment: Unduh dan instal Node.js (versi LTS disarankan) pada laptop yang akan dijadikan server utama pangkalan data rumah sakit.
  2. Ekstraksi Source Code: Salin folder proyek sistem antrian ke direktori laptop server.
  3. Instalasi Dependensi: Buka terminal (Command Prompt/PowerShell) di dalam folder proyek tersebut, lalu jalankan perintah berikut untuk menginstal modul Express dan Socket.io: 
    npm install express socket.io
  4. Mengetahui IP Lokal Server: Buka terminal baru dan ketik perintah ipconfig (Windows) atau ifconfig (Mac/Linux). Cari bagian IPv4 Address pada adaptor jaringan Wi-Fi/Ethernet yang aktif (Contoh: 192.168.1.15). Catat nomor IP ini.
  5. Menjalankan Server: Eksekusi program backend dengan mengetik perintah berikut di terminal: 
    node server.js
    Pastikan muncul pesan log yang menyatakan server anti-blackout aktif di port 5500.

Bagian B: Konfigurasi Keamanan Jaringan (Penting!)

Agar laptop atau HP orang lain dapat mengakses server, sistem Firewall Windows pada laptop server harus mengizinkan lalu lintas port 5500, atau matikan Windows Defender Firewall untuk jaringan lokal (Private Network) secara sementara selama simulasi berlangsung guna menghindari error Connection Timed Out.

Bagian C: Menghubungkan Perangkat Client (Laptop/HP Orang Lain)

Pastikan semua perangkat client terhubung ke router Wi-Fi atau access point lokal yang sama dengan laptop server. Perangkat client tidak perlu menginstal Node.js, mereka cukup membuka web browser dan mengakses URL berikut menggunakan IP Lokal Server yang telah dicatat sebelumnya:

  • Kios Pengambilan Tiket Pasien:
    http://IP_SERVER_ANDA:5500/public/kios.html
  • Layar TV Monitor Ruang Tunggu:
    http://IP_SERVER_ANDA:5500/public/monitor.html
  • Portal Utama / Dashboard HP Dokter:
    http://IP_SERVER_ANDA:5500
