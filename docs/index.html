Sistem Antrian LAN Terdistribusi RS Waluyo (Anti-Blackout Version)

1. Gambaran Umum Sistem

Sistem Antrian LAN Terdistribusi RS Waluyo dirancang khusus untuk mengotomatisasi seluruh proses administrasi penomoran antrian pasien di poliklinik secara real-time tanpa menggunakan koneksi internet luar (murni menggunakan jaringan intranet / LAN).

Tujuan dan Manfaat

  • Tujuan: Mempercepat alur pelayanan loket rumah sakit dan mengintegrasikan antrian lintas poliklinik secara terpusat.
  • Manfaat: Menghilangkan penumpukan antrean fisik, meningkatkan transparansi pameran nomor lewat TV monitor ruang tunggu, serta mempermudah dokter memanggil pasien jarak jauh via HP pribadi.

Target Pengguna

  1. Pasien: Pengambil nomor tiket mandiri pada unit kios di lobi utama.
  2. Dokter / Perawat: Pemegang kendali panggilan antrian poliklinik melalui dashboard HP medis.
  3. Pengunjung Umum: Pemantau status antrian pada layar TV utama di ruang tunggu.

Fitur Unggulan

  • Kios Tiket Multi-Poli: Penomoran otomatis berbasis prefiks poli (Umum: A, Gigi: B, Anak: C).
  • Real-Time WebSocket Sync: Transmisi data instan antar perangkat memanfaatkan library Socket.io.
  • Voice Announcement System: Suara bel otomatis ("Ding Dong") yang ditiupkan langsung dari browser TV ruang tunggu.
  • Anti-Blackout Storage: Keamanan data tingkat tinggi menggunakan modul fs (File System) Node.js yang menyimpan data antrian ke dalam harddisk fisik server setiap kali ada perubahan, melindunginya dari risiko mati lampu.
  • Auto-Reset System: Reset berkala data antrian menjadi 000 secara otomatis setiap kali mendeteksi tanggal baru di hari berikutnya.

2. Panduan Penggunaan Fitur Secara Terurut

Sistem ini beroperasi mengikuti urutan sekuensial yang saling terintegrasi antar perangkat sebagai berikut:

Langkah 1: Pengambilan Tiket (Kios Pasien)

Pasien menggunakan HP/Tablet di lobi utama yang mengarah ke alamat URL http://IP_SERVER:5500/public/kios.html. Pasien memilih poliklinik yang dituju, dan layar akan menampilkan nomor tiket resmi (misal: B-001).

Langkah 2: Pemantauan Layar Tunggu (TV Monitor)

Petugas menyalakan browser Smart TV di ruang tunggu dan membuka alamat URL http://IP_SERVER:5500/public/monitor.html. Petugas wajib melakukan klik satu kali pada area layar mana saja untuk memicu aktivasi sistem audio bel browser.

Langkah 3: Pemanggilan Pasien (HP Dokter)

Dokter membuka HP pribadinya, mengakses portal utama http://IP_SERVER:5500, dan masuk ke Panel Medis sesuai poliklinik tugasnya. Ketika dokter menekan tombol "Panggil Selanjutnya", server memperbarui status data di harddisk, merubah nomor di TV ruang tunggu, dan langsung menyalakan bunyi bel panggilan.

Langkah 4: Penyelesaian Sesi

Begitu penanganan medis selesai, dokter menekan tombol "Selesai Penanganan" pada HP miliknya agar indikator loket di monitor TV berubah menjadi status SELESAI dan siap memanggil pasien selanjutnya.
